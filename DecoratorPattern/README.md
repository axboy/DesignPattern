# 装饰器模式

    我想做，但不能做，需要一个团队帮我做

* 意图

    动态地给一个对象添加一些额外的职责。就增加功能来说，装饰器模式相比生成子类更为灵活。
    
* 何时使用

    在不想增加很多子类的情况下扩展类
    
* 优点

    装饰类和被装饰类可以独立发展，不会相互耦合。装饰模式是继承的一个替代模式。
    
* 缺点

    多层装饰比较复杂
